# PRD (Product Requirements Document) — 시료 주문 시스템

## 개요

### 생산 라인
- 공장에서 시료 하나를 생산하는 설비 흐름
- 하나의 생산 라인은 시료를 하나씩 생산
- 주문이 들어온 시료에 대해서만 생산

### 시스템 운영 방식
- 콘솔 기반으로 동작
- 담당자가 직접 명령을 입력
- 시료를 등록하고 주문을 처리

### 사용자 역할
- 고객 주문 담당자
- 생산 담당자

---

## 주문 상태 흐름

| 상태 | 설명 |
|------|------|
| `RESERVED` | 주문 접수 (최초 주문 시 부여) |
| `REJECTED` | 주문 거절 |
| `PRODUCING` | 재고 부족으로 주문 승인 완료 및 생산 중 → 생산 완료 후 `CONFIRMED`로 전환 |
| `CONFIRMED` | 재고 충분, 주문 승인 완료 및 출고 대기 중 |
| `RELEASE` | 출고 완료 |

- 최초 주문 시 → `RESERVED` 상태

## 생산 담당자 기능

### 1. 시료 관리
새로운 시료 등록, 목록 조회, 이름 검색 (CRUD)

#### 시료 등록
- 새로운 시료를 시스템에 추가
- 속성 값: 시료 ID, 이름, 평균 생산시간, 수율

#### 시료 조회
- 등록된 모든 시료 목록을 확인
- 현재 재고 수량도 함께 표시

#### 시료 검색
- 이름 등 속성으로 특정 시료를 검색

---

### 2. 주문 승인/거절
접수된 주문(`RESERVED`) 목록을 확인하고, 특정 주문에 대하여 승인 또는 거절 처리

#### 접수된 주문 목록
- `RESERVED` 상태의 주문 목록 표시

#### 주문 승인
접수된 특정 주문에 대해 승인 — 재고 상황에 따라 자동 처리:

- **재고 충분** → 즉시 `CONFIRMED` 상태로 전환
- **재고 부족** → 생산 라인에 자동 등록, 주문 상태를 `PRODUCING`으로 전환

#### 주문 거절
- 접수된 특정 주문에 대해 거절
- 즉시 `REJECTED` 상태로 전환

---

### 3. 생산 라인
현재 생산 중인 시료 및 대기 중인 생산 큐 확인

#### 생산 계획
주문량에 대한 부족분을 생산하되, 수율 및 오차를 고려하여 시료를 생산:

- **실 생산량**: `부족분 / 수율 * 0.9`
- **총 생산 시간**: `평균 생산시간 × 실 생산량`
- 생산 완료 시 주문 상태: `PRODUCING` → `CONFIRMED`

#### 생산 현황 표시
- 현재 생산 중인 시료에 대한 정보 표시
- 표시 정보 수준은 자율적으로 결정 (예: 주문 정보, 현재까지의 생산량 등)

#### 대기 주문 확인
- 생산 라인의 대기열(생산 큐) 활용
- 생산 작업을 대기 중인 목록 출력
- **스케줄링 전략**: FIFO

## 주문 담당자 기능

### 1. 시료 주문 — 고객 주문 접수

#### 시료 예약
- 고객이 원하는 시료와 수량을 주문
- 주문 시점에서 상태: `RESERVED`

#### 예약 시 입력 값
- 시료 ID
- 고객명
- 주문 수량

---

### 2. 모니터링
상태별 주문 수 및 시료별 재고 현황 확인

담당자가 현재 시스템의 상태를 한눈에 파악할 수 있도록 구성

#### 주문량 확인
- 현재 상태별(`RESERVED` / `CONFIRMED` / `PRODUCING` / `RELEASE`) 목록 확인
- `REJECTED`는 유효한 주문이 아니므로 제외

#### 재고량 확인
- 각 시료별 현재 재고 수량 확인
- 주문 대비 재고 수량에 따른 상태 표기:

| 상태 | 조건 |
|------|------|
| 여유 | 주문 대비 재고 충분 |
| 부족 | 주문 대비 재고 수량 부족 |
| 고갈 | 수량이 0인 상태 |

---

### 3. 출고 처리
- 재고가 충분해진 `CONFIRMED` 주문에 대하여 출고를 처리
- 특정 주문에 대해 출고를 실행
- 주문 상태가 `RELEASE`로 전환
